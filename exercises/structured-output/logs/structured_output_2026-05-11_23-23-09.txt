  HOW IT WORKS:
----------------------------------------------------------------------

  TECHNIQUE 1: Explicit Format Instructions in User Message
  ─────────────────────────────────────────
  Put strict format rules directly in the user message:
  'Output ONLY raw JSON. No markdown. No backticks. Start with { end with }.'
  This constrains the model's output format before it even starts.

    messages = [
      {"role": "user", "content": "Output ONLY raw JSON..."}
    ]

  NOTE: On some models, system-role messages cause reasoning tokens
  to consume the output. Putting instructions in the user message
  works more reliably across different API providers.

  TECHNIQUE 2: Stop Sequences
  ─────────────────────────────────────────
  You specify a string that triggers IMMEDIATE stop of generation.
  When the model generates that string, output is cut off right there.
  Safety net in case the model tries to add commentary.

    response = client.chat.completions.create(
        ..., stop=["```"]  ← STOP SEQUENCE
    )

  ANTHROPIC-SPECIFIC: Assistant Message Prefilling
  ─────────────────────────────────────────
  On Anthropic's Claude API, you can add a fake assistant message
  to make Claude think it already started writing in a format:

    messages = [
      {"role": "user", "content": "Generate JSON..."},
      {"role": "assistant", "content": "```json\n"},  ← PREFILL (Anthropic only)
    ]

  COMBINED: System prompt constrains format, stop sequence cuts extras.
  You get EXACTLY the data you want — nothing before, nothing after.

======================================================================
  SUMMARY: WHEN TO USE THESE TECHNIQUES
======================================================================

  | Use Case                    | System Prompt              | Stop Sequence |
  |-----------------------------|----------------------------|---------------|
  | JSON for API pipelines      | Only JSON, no markdown     | ```           |
  | Python code for execution   | Only code, no comments     | ```           |
  | Shell commands              | Only commands, one/line    | \n\n          |
  | CSV data for import         | Only CSV rows              | ```           |
  | YAML config                 | Only YAML, no markdown     | ```           |

  NOTE: On Anthropic Claude API, also use assistant message prefilling
  (add fake assistant message like '```json\n' to make it continue in
  that format). On OpenAI-compatible APIs, system prompts work better.

  KEY INSIGHT:
  Without these techniques: output needs manual cleanup before use.
  With these techniques: output goes directly into your pipeline.
