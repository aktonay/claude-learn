  TECHNIQUE 1: Assistant Message Prefilling
  ─────────────────────────────────────────
  You add a fake 'assistant' message before the API responds.
  The model thinks it ALREADY started writing, so it CONTINUES
  in that format instead of starting fresh with commentary.

    messages = [
      {"role": "user", "content": "Generate JSON..."},
      {"role": "assistant", "content": "```json\n"},  ← PREFILL
    ]

  TECHNIQUE 2: Stop Sequences
  ─────────────────────────────────────────
  You specify a string that triggers IMMEDIATE stop of generation.
  When the model generates that string, output is cut off right there.

    response = client.chat.completions.create(
        ..., stop=["```"]  ← STOP SEQUENCE
    )

  COMBINED: Prefill opens the format, stop sequence closes it.
  You get EXACTLY the data you want — nothing before, nothing after.
