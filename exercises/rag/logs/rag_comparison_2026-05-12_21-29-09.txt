  THE RAG PIPELINE:
----------------------------------------------------------------------

  Step 1: CHUNK the document into smaller pieces
  Step 2: EMBED each chunk (text -> numbers via VoyageAI)
  Step 3: STORE embeddings in a vector database
  Step 4: EMBED the user's query
  Step 5: SEARCH for most similar chunks
  Step 6: GENERATE answer from top chunks + query via LLM

  THREE SEARCH STRATEGIES COMPARED:

  +------------+--------------------------------+--------------------------------+
  | Strategy   | Strength                       | Weakness                       |
  +------------+--------------------------------+--------------------------------+
  | Semantic   | Understands MEANING/CONTEXT    | Misses exact term matches      |
  | (Vector)   | 'accomplish' -> project work   | 'INC-2023-Q4-011' -> ?         |
  +------------+--------------------------------+--------------------------------+
  | Lexical    | Finds EXACT TERM matches       | No understanding of synonyms   |
  | (BM25)     | 'INC-2023-Q4-011' -> found!    | 'accomplish' != 'achieved'     |
  +------------+--------------------------------+--------------------------------+
  | Hybrid     | BEST of both via RRF fusion    | More complex to implement      |
  | (Retriever)| Meaning + exact matches        | Two indexes to maintain        |
  +------------+--------------------------------+--------------------------------+

  RECIPROCAL RANK FUSION (RRF):
  RRF_score(doc) = sum( 1 / (k + rank) ) for each index
  Merges ranked lists without needing score normalization.
  Docs that rank well in MULTIPLE indexes get boosted.

  CONCLUSION:

  Semantic search excels at understanding INTENT:
    'What did the team accomplish?' -> finds project work sections
    Even though 'accomplish' never appears in the document.

  BM25 excels at EXACT TERM matching:
    'INC-2023-Q4-011' -> finds the exact section with that ID
    Semantic search might miss this because the ID is opaque.

  Hybrid (RRF) gets the BEST of both:
    Documents that rank well in BOTH indexes get boosted.
    This is the standard approach in production RAG systems.

  WHY HYBRID IS BETTER (mathematically):
    RRF_score = sum( 1/(k + rank_i) ) for each index i
    A doc at rank 1 in BOTH indexes:  1/61 + 1/61 = 0.0328
    A doc at rank 1 in ONE index:      1/61 + 1/62 = 0.0322
    Consistently relevant docs always win over one-hit wonders.
