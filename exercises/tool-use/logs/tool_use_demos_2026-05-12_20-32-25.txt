  HOW TOOL CALLING WORKS (OpenAI-compatible API):
----------------------------------------------------------------------

  THE FLOW:
  1. You send a user message + tool schemas to the model
  2. Model decides if it needs external data to answer
  3. If yes: model returns finish_reason='tool_calls'
     with a tool_calls list: [{id, function: {name, arguments}}]
  4. Your code executes each tool function
  5. You send tool results back as: {role: 'tool', tool_call_id, content}
  6. Model uses results to either:
     a. Call more tools (loop back to step 2)
     b. Give a final text answer (finish_reason='stop')

  THE 3 TOOLS:
  ┌─────────────────────────┬────────────────────────────────────┐
  │ Tool                    │ Purpose                            │
  ├─────────────────────────┼────────────────────────────────────┤
  │ get_current_datetime    │ Returns current date/time          │
  │ add_duration_to_datetime│ Adds days/hours/min to a datetime  │
  │ set_reminder            │ Records reminder with target time  │
  └─────────────────────────┴────────────────────────────────────┘

  WHY TOOLS ARE NEEDED:
  - Model doesn't know EXACT current time → tool 1 fixes this
  - Model struggles with complex date math → tool 2 fixes this
  - Model can't actually SET anything → tool 3 fixes this

  KEY INSIGHT:
  The model decides WHAT tools to call and in WHAT ORDER.
  Your job is to provide the implementations.
  This is the 'agentic loop' pattern used in all AI agent systems.
